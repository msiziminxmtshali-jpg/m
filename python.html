Project Overview

This project focuses on analyzing a retail dataset to uncover sales trends, customer purchasing behavior, and inventory turnover. Using SQL (PostgreSQL), I cleaned the data and performed aggregate analysis to help stakeholders understand which regions perform best.

Tools Used

  • SQL (Joins, CTEs, Window Functions)
  • Microsoft Excel (Data cleaning)
  • Power BI (Visualization)

Key SQL Code

To determine which products drive the most revenue, I joined the Orders and Products tables and grouped by category.

SELECT p.product_name, 
       SUM(o.quantity * p.price) AS total_revenue
FROM orders o
JOIN products p ON o.product_id = p.product_id
GROUP BY p.product_name
ORDER BY total_revenue DESC
LIMIT 5;

Conclusion

Based on this analysis, sales peak significantly in Q4, suggesting a need for increased stock levels in October. The visual data presented in the slideshow above highlights the correlation between marketing spend and regional sales spikes.
